Documentation

WriteKit is an MCP-native blogging platform. You manage your blog entirely through an AI assistant — no editor, no dashboard. Connect, write, publish.

Get started

1. Add the MCP server

claude mcp add --transport http writekit https://writekit.dev/mcp

Add to .cursor/mcp.json:

{
  "mcpServers": {
    "writekit": {
      "url": "https://writekit.dev/mcp"
    }
  }
}

Add to your Windsurf MCP config:

{
  "mcpServers": {
    "writekit": {
      "url": "https://writekit.dev/mcp"
    }
  }
}

Add to .vscode/mcp.json:

{
  "servers": {
    "writekit": {
      "type": "http",
      "url": "https://writekit.dev/mcp"
    }
  }
}
opencode mcp add --transport http writekit https://writekit.dev/mcp

Point any Streamable HTTP MCP client at:

https://writekit.dev/mcp

2. Authenticate

A browser window opens automatically on first tool use. Sign in with GitHub, Google, or Discord. This only happens once — done.

3. Create your blog

After signing in, visit your dashboard to pick a subdomain. Your blog lives at yourname.writekit.dev.

4. Start writing

Tell your AI assistant what to write. It handles drafting, formatting, and publishing through MCP tools.

Example prompts

Copy these directly into Claude. They work as-is once you're connected.

"Write a blog post about Go concurrency patterns"
Creates a draft with markdown content, returns a preview URL.
"Show me all my draft posts"
Lists all drafts with titles, IDs, and dates.
"Publish my post about Go concurrency"
Sets the post to published, returns the live URL.
"Update my latest post — add a section about error handling"
Fetches the post, appends content, regenerates HTML.
"Are there any new comments?"
Lists recent comments across all your posts.
"Delete the spam comment from user123"
Removes the comment by ID.
"Change my blog title to 'Ship Fast, Write Often'"
Updates the blog title in settings.
"Search my posts for anything about databases"
Full-text search across titles and content.

MCP tools

These tools are available to your AI assistant when connected to WriteKit.

ToolDescription
create_postCreate a new draft post with markdown content
update_postUpdate any field of an existing post (partial updates)
delete_postPermanently delete a post
publish_postPublish a draft — returns the live URL
unpublish_postRevert a published post to draft
list_postsList posts, filter by status or tag
get_postGet full post content by ID or slug
search_postsFull-text search across all posts
list_commentsList comments on a post
delete_commentDelete a comment by ID
get_settingsGet current blog settings
update_settingsUpdate blog title, description

Markdown

Posts are written in markdown and rendered with Goldmark. Supported extensions:

  • GitHub Flavored Markdown (tables, strikethrough, autolinks, task lists)
  • Footnotes
  • Auto heading IDs
  • Syntax highlighting for code blocks (use language tags: ```go, ```python, etc.)
  • Raw HTML

Blog features

Every blog at yourname.writekit.dev includes:

  • Index page — latest published posts
  • Post pages — /posts/{slug}
  • Tag pages — /tags/{tag}
  • Search — /search?q=query
  • RSS feed — /feed.xml
  • Comments — built-in form on every post
  • Preview URLs — shareable expiring links for drafts

Architecture

  • Per-tenant SQLite — each blog has its own database file (WAL mode)
  • PostgreSQL — platform data (users, sessions, OAuth, billing)
  • No Redis, no workers — single binary, in-process event bus
  • MCP transport — Streamable HTTP with OAuth 2.0 + PKCE
