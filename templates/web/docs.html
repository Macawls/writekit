Get started

WriteKit plugs into your AI assistant. You tell it what to write, it publishes to your site. No clicking around a dashboard, no copy-pasting Markdown into forms.

The setup is four steps. Takes about two minutes.

1. Connect your AI

Pick your tool below and run the command (or paste the config). This tells your AI about WriteKit. 

claude mcp add --transport http writekit https://writekit.dev/mcp

Add to .cursor/mcp.json:

{
  "mcpServers": {
    "writekit": {
      "url": "https://writekit.dev/mcp"
    }
  }
}

Add to your Windsurf MCP config:

{
  "mcpServers": {
    "writekit": {
      "url": "https://writekit.dev/mcp"
    }
  }
}

Add to .vscode/mcp.json:

{
  "servers": {
    "writekit": {
      "type": "http",
      "url": "https://writekit.dev/mcp"
    }
  }
}

Add to opencode.json:

{
  "mcp": {
    "writekit": {
      "type": "remote",
      "url": "https://writekit.dev/mcp"
    }
  }
}

Point any Streamable HTTP MCP client at:

https://writekit.dev/mcp

2. Sign in

The first time your AI tries to use WriteKit, a browser tab opens. Pick how you want to sign in — GitHub, Google, Discord, or email. That's it.

3. Pick a subdomain

After signing in, you'll pick a name for your site. Your pages will live at yourname.writekit.dev. You can bring a custom domain later.

4. Start writing

You're done. Now just talk to your AI.

Try it: Ask your AI — "Write a short post about why I love Go, and publish it."

Your AI will create the page, show you a preview link, and publish once you're happy with it. You never touch a form.

The writing loop

Every page follows the same flow:

  1. Ask. Tell your AI what to write. It creates a draft and returns a preview link.
  2. Preview. Open the link — it reloads automatically as the AI edits.
  3. Iterate. "Make it shorter." "Add a section on X." "Change the tone." The AI edits, the preview updates.
  4. Publish. Say "publish it." Your page goes live at yourname.writekit.dev/{slug}.

Every edit is saved as a version. If something got worse, add ?v=N to the preview URL to look at an earlier one.

Core concepts

Pages

A page is one piece of writing. A post, a doc, a recipe — whatever. Pages start as drafts, go live when you publish them, and get a URL based on their slug.

Collections

Collections are folders. Group related pages together — a tutorial series, a "docs" collection, a changelog. Pages can also live on their own, outside any collection.

Collections can be ordered manually (good for docs in a learning order) or by date (good for changelogs).

Visibility

Every page has a visibility setting:

  • Public — anyone can see it. Shown in your index and search.
  • Unlisted — only people with the link. Hidden from the index and search.
  • Private — only signed-in team members. Everyone else gets a 404.

A draft isn't visible at all — visibility only kicks in once you publish.

Teams

Every site has a team. Invite people by email. Three roles:

  • Owner — full control: billing, settings, everyone, everything.
  • Editor — can write, publish, and delete. Can see private pages.
  • Viewer — read-only. Can see private pages but can't change them.

URLs

  • Standalone page: yourname.writekit.dev/{slug}
  • Collection: yourname.writekit.dev/{collection}
  • Page in collection: yourname.writekit.dev/{collection}/{slug}
  • Raw Markdown: add .md to any page URL
  • Search: yourname.writekit.dev/search?q={query}
  • Preview (for drafts): yourname.writekit.dev/preview/{token}

MCP tools

Tools your AI can call once WriteKit is connected. You don't call these directly — your AI does, when you ask for things.

Pages

Tool What it does
create_page Make a new draft
update_page Change anything on an existing page
delete_page Delete for good
publish_page Go live — returns the URL
unpublish_page Move back to draft
list_pages List pages, filter by status/tag/collection
get_page Fetch a page's full content
append_to_page Add to the end without resending everything
search_pages Full-text search

Collections

Tool What it does
create_collection Make a new collection
update_collection Rename, re-slug, change sort order
delete_collection Delete (pages inside become standalone)
list_collections List all with page counts
get_collection Details + pages inside
reorder_pages Set manual page order

Settings & teams

Tool What it does
get_settings Current site settings
update_settings Change title, description
list_members Show all team members
invite_member Invite someone by email (owner only)
remove_member Remove a member (owner only)
update_member_role Change a member's role (owner only)

MCP resources

Read-only context your AI can pull at any time — useful for "what's going on with my site right now?" questions.

URI What you get
writekit://site/stats Page counts, collection counts
writekit://site/settings Current site settings
writekit://site/recent-pages Last 10 published
writekit://site/drafts All drafts
writekit://site/collections All collections with page counts

Markdown

Pages are written in Markdown. On top of the basics, you get:

  • GitHub Flavored Markdown — tables, strikethrough, auto-links, task lists
  • Syntax highlighting — fenced code blocks with a language tag (```go, ```python, etc.)
  • Callouts — > [!NOTE], > [!TIP], > [!WARNING], > [!DANGER]
  • Embeds — <embed src="url" /> for YouTube, Spotify, SoundCloud, Twitter/X, GitHub Gists
  • Diagrams — ```d2 blocks render as architecture diagrams
  • Footnotes, auto heading IDs, and raw HTML when you need it
