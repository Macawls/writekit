Microsoft's Acquisition of GitHub

draft · 7 min read · updated 2 min ago

In June 2018, Microsoft announced it would acquire GitHub for $7.5 billion in stock. The deal closed that October. At the time, the reaction from developers was mixed at best — GitLab saw a spike in repository imports within hours of the announcement, and forums filled with predictions that Microsoft would slowly degrade the platform.

Years on, the acquisition is one of the cleaner examples of a large tech company buying a developer-facing product and not breaking it.

Years on, the deal stands as one of the cleaner examples of a large platform-company acquiring a developer product and leaving it intact — arguably the cleanest in the last decade.

The deal

  • Price: $7.5 billion, paid in Microsoft stock.
  • Announced: June 4, 2018.
  • Closed: October 26, 2018.

Microsoft was not the obvious buyer historically. Under Steve Ballmer, the company had called Linux a cancer. By 2018 that posture had shifted: Microsoft was already the largest contributor to open source on GitHub by employee count.

Why Microsoft wanted it

GitHub was the default home for source code. Owning it meant:

  • A direct relationship with tens of millions of developers.
  • A distribution channel for Azure, Visual Studio, and later Copilot.
  • Data — public repositories at a scale no one else had — that would later prove central to training code models.

The price looked high relative to GitHub's revenue at the time, but cheap relative to the strategic position they'd be buying.

What changed

Most of the visible changes after the acquisition were additions, not removals. Free private repos rolled out in early 2019. GitHub Actions launched the same year. npm joined in 2020. Copilot landed in 2021.

on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: go test ./...

The platform's core — git hosting, pull requests, issues — was left alone.

It's a useful reminder that "company A buys company B" doesn't always end the way the worst-case forum threads predict.
